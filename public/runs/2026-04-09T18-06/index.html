PyTorch Foundation Adopts Safetensors as Official Contributed Project

Other

The PyTorch Foundation has formally adopted Safetensors, Hugging Face's secure model serialization format, as a contributed project. The move standardizes a safer alternative to pickle-based model loading across the ML ecosystem.

The PyTorch Foundation has announced that Safetensors, the secure tensor serialization format originally developed by Hugging Face, is now an official contributed project under the Foundation's governance. This formalizes what has been a de facto industry migration away from Python's pickle format for model storage and loading.

Safetensors was created to address a well-known security vulnerability in the ML ecosystem: pickle files can execute arbitrary code when loaded, creating a vector for supply-chain attacks. As models are increasingly shared across organizations and downloaded from public repositories like Hugging Face Hub, this attack surface has grown dramatically. Safetensors provides a format that is inherently safe to load — no code execution possible — while also offering faster loading times and lower memory overhead.

The PyTorch Foundation's formal adoption is significant because it signals institutional commitment to making Safetensors a standard rather than just a popular library. Under Foundation governance, the format will benefit from broader contributor access, more rigorous maintenance, and alignment with PyTorch's roadmap. This should accelerate adoption among organizations that were hesitant to depend on a format governed by a single company.

For ML engineers, this is largely a validation of existing best practices — most teams have already migrated to Safetensors for model distribution. But the Foundation adoption makes it easier to justify in enterprise contexts where governance and supply-chain security are formal requirements. Model registries and deployment pipelines that still support pickle loading should prioritize deprecation.

The bottom line: a critical piece of ML infrastructure security is now under proper community governance. This is how open-source standards should evolve.
